# 🎬 YT Downloader x KUDASAI

Расширение для **Google Chrome** и **Яндекс Браузера** + локальный сервер для скачивания видео с YouTube в любом качестве.

Одна кнопка **«Скачать»** на странице YouTube — выбираешь качество, ждёшь, получаешь файл в загрузках браузера (`Ctrl+J`).

## 🚀 Быстрый старт

### Вариант 1 — готовая программа (.exe), рекомендуется

1. Скачай архив со страницы **[Releases](../../releases)** (`YT-Downloader.zip`) и распакуй
2. Запусти `YT Downloader.exe`
   - Кнопка **Run** станет **Stop** (красной) — сервер запущен
   - Окно можно закрыть — программа останется в трее рядом с часами
3. Убедись, что установлен **ffmpeg** (см. [раздел ниже](#ffmpeg))
4. Установи расширение в браузер (см. [ниже](#установка-расширения))
5. Открой YouTube → кнопка **«Скачать»** рядом с лайком — готово!

### Вариант 2 — из исходников

Нужен **Python 3.9+** (при установке отметь **Add Python to PATH**).

```bash
pip install PySide6 flask flask-cors yt-dlp requests
python launcher.py
```

Лаунчер сам поднимет встроенный сервер в фоновом потоке.

Без лаунчера — только сервер:
```bash
pip install flask flask-cors yt-dlp
python server/server.py
```

## 🎛️ Настройки лаунчера

| Переключатель | Описание |
|---|---|
| **Запускать сервер при старте программы** | Сервер поднимается автоматически при запуске лаунчера (включён по умолчанию) |
| **Запускать при включении Windows** | Программа тихо стартует в трее при загрузке системы |
| **Внешний сервер (опционально)** | Использовать свой `server.py` / `server.exe` вместо встроенного — только если нужен кастомный сервер |

После изменения переключателей нажми **Сохранить**.

### Автозапуск в трее

При включении «Запускать при включении Windows» создаётся ярлык в папке автозагрузки:
```
%APPDATA%\Microsoft\Windows\Start Menu\Programs\Startup\YTDownloaderLauncher.bat
```
Команда: `"pythonw.exe" "путь\к\launcher.py" --tray`

## 🎥 Установка расширения

### Google Chrome
1. Открой `chrome://extensions`
2. Включи **Режим разработчика** (переключатель справа вверху)
3. Нажми **Загрузить распакованное расширение**
4. Выбери папку `extension` из проекта

### Яндекс Браузер
1. Открой `browser://extensions`
2. Включи **Режим разработчика**
3. Нажми **Загрузить распакованное расширение**
4. Выбери папку `extension` из проекта

## 📥 Использование

1. Запусти `YT Downloader.exe` (или `python launcher.py`) — дождись, пока кнопка станет красной и покажет **Stop**
2. Открой любое видео или Shorts на YouTube
3. Нажми кнопку **Скачать** рядом с лайком

## 🔧 ffmpeg

Нужен для склейки видео и аудио потоков в один MP4-файл. Без него скачивание **не будет работать** — `ffmpeg` не входит ни в лаунчер, ни в `server.py`, его нужно поставить отдельно один раз.

### Через winget (Windows 10/11)
```bash
winget install Gyan.FFmpeg
```

### Вручную
1. Скачай сборку с [gyan.dev](https://www.gyan.dev/ffmpeg/builds/)
2. Распакуй архив
3. Добавь папку `bin` в переменную окружения `PATH`

### Проверка
Открой командную строку и выполни:
```bash
ffmpeg -version
```
Если увидишь информацию о версии — всё готово.

## ✅ Проверка сервера

Пока сервер запущен, в браузере должен открываться:
```
http://127.0.0.1:5001/ping
```

Если всё хорошо — вернётся JSON:
```json
{"ok": true, "temp_dir": "C:\\Users\\...\\Downloads\\YouTube\\.tmp_browser_downloads"}
```

Сервер слушает **только** `127.0.0.1` — он недоступен из интернета или локальной сети.

## 🔐 Безопасность

- Сервер работает **исключительно локально** (`127.0.0.1:5001`)
- Никакие данные не отправляются на внешние серверы
- Все зависимости — открытые проекты с открытым исходным кодом:
  - [Flask](https://flask.palletsprojects.com/)
  - [yt-dlp](https://github.com/yt-dlp/yt-dlp)
  - [PySide6 (Qt)](https://www.qt.io/product/qt6)
- Расширение работает только на доменах YouTube

## 🤔 Почему гибрид: расширение + локальный сервер?

Обычное браузерное расширение не умеет надёжно скачивать YouTube в хорошем качестве, потому что:

- Видео и аудио часто идут **раздельными потоками**
- Для склейки нужен `ffmpeg`, который недоступен из браузера
- Ссылки и форматы YouTube **постоянно меняются**

Поэтому интерфейс даёт расширение, а реальную загрузку делает локальный сервер с `yt-dlp`.

## ❓ Частые проблемы

| Проблема | Решение |
|---|---|
| В окне лаунчера горит **«Офлайн»** | Нажми **Run** — сервер ещё не запущен |
| Кнопка **«Скачать»** не появляется на YouTube | Перезагрузи страницу, проверь что расширение включено в `chrome://extensions` |
| **Ошибка при склейке видео** | Не установлен `ffmpeg` — см. [раздел выше](#ffmpeg) |
| `launcher.py` не запускается | Установи зависимости: `pip install PySide6 flask flask-cors yt-dlp requests` |
| `server.py` не запускается из исходников | Установи зависимости сервера: `pip install flask flask-cors yt-dlp` |
| Сервер запускается, но файл не скачивается | Проверь что расширение загружено в браузер и кнопка «Скачать» видна на странице |

## 🛠️ Сборка .exe

```bash

    --name "YT Downloader" --icon "build/icon.ico" ^
    --collect-all yt_dlp --collect-all flask_cors ^
    launcher.py
```

### Сервер (отдельный .exe)
```bash
pyinstaller --noconfirm --clean --onefile --windowed ^
    --name "YT Downloader Server" --icon "build/icon.ico" ^
    --collect-all yt_dlp --collect-all flask_cors ^
    server/server.py
```
